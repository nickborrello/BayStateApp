---
description: Commit changes to git with a conventional commit message
---

1. Check the current git status to identify modified files.
   Command: `git status`

2. View the changes to understand what is being committed.
   Command: `git diff`

3. Add the relevant files to the staging area.
   Command: `git add <files>`

4. Generate a conventional commit message based on the changes (e.g., `feat:`, `fix:`, `refactor:`) and commit.
   Command: `git commit -m "<message>"`
